# Product Context

## Why this project exists

Developers often run many local services at once and lose track of which port belongs to which process. Existing tools are fragmented, terminal-heavy, or too low-level for fast day-to-day inspection.

## User problems

- Identifying what is bound to a port takes too many steps.
- Process metadata can be inconsistent or inaccessible on Windows.
- Developers need a single place to inspect local services without switching between terminals, task manager, and browser tools.

## MVP experience goals

- Show a trustworthy list of active local ports quickly.
- Surface the most useful process details without blocking on inaccessible metadata.
- Make refresh, search, filtering, and sorting feel immediate and reliable.
- Keep the UI calm, readable, and optimized for desktop use.
